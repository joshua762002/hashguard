# HashGuard — Setup Guide (XAMPP)

A complete PHP/MySQL web application about password hashing and security.

---

## Requirements

- XAMPP (PHP 7.4+ and MySQL 5.7+ / MariaDB 10.3+)
- A modern web browser

---

## Step-by-Step Installation

### Step 1 — Download & Place Files

1. Copy the entire `hashguard` folder into your XAMPP `htdocs` directory:
   ```
   C:\xampp\htdocs\hashguard\        ← Windows
   /Applications/XAMPP/htdocs/hashguard/  ← macOS
   /opt/lampp/htdocs/hashguard/      ← Linux
   ```

### Step 2 — Start XAMPP

1. Open the **XAMPP Control Panel**.
2. Start **Apache** and **MySQL**.

### Step 3 — Create the Database

1. Open your browser and go to: `http://localhost/phpmyadmin`
2. Click **"New"** in the left sidebar to create a new database.
3. Name it `hashguard_db` and click **Create**.
4. Click on `hashguard_db` in the left sidebar.
5. Click the **"Import"** tab at the top.
6. Click **"Choose File"** and select `hashguard/database.sql`.
7. Click **"Go"** at the bottom. You should see a success message.

### Step 4 — Configure Database Connection (if needed)

Open `hashguard/config/db.php` and verify the settings match your XAMPP setup:

```php
define('DB_HOST', 'localhost');
define('DB_USER', 'root');    // default XAMPP username
define('DB_PASS', '');        // default XAMPP password is empty
define('DB_NAME', 'hashguard_db');
```

> **Note:** If you set a MySQL root password in XAMPP, update `DB_PASS` accordingly.

### Step 5 — Open the Application

Go to: `http://localhost/hashguard/`

You will be redirected to the login page.

## Default Credentials

| Role  | Email                  | Password     |
|-------|------------------------|--------------|
| Admin | admin@hashguard.com    | Admin@1234   |
| User  | Register a new account | (your choice)|

> **Important:** The admin password hash in `database.sql` uses bcrypt with cost 12.
> PHP's `password_verify()` will correctly verify it regardless of whether
> the hash prefix is `$2b$` (Python-generated) or `$2y$` (PHP-generated) —
> they are interchangeable in PHP 7+.

## Security Features

| Feature | Implementation |
|---------|---------------|
| Password hashing | `password_hash()` with bcrypt, cost 12 |
| Password verification | `password_verify()` — constant-time safe |
| SQL injection prevention | PDO prepared statements throughout |
| Session fixation prevention | `session_regenerate_id(true)` on login |
| Admin page protection | `requireAdmin()` guard at top of each admin file |
| Session hijacking mitigation | Sessions destroyed on logout (cookie + server) |
| XSS prevention | `htmlspecialchars()` on all user output |

## How Passwords Are Handled

```
Registration:
  User types "mySecret123"
       ↓
  password_hash("mySecret123", PASSWORD_BCRYPT, ['cost' => 12])
       ↓
  Stored in DB: "$2y$12$abc...xyz" (60-char bcrypt hash)
  Plain text is NEVER stored.

Login:
  User types "mySecret123"
       ↓
  password_verify("mySecret123", "$2y$12$abc...xyz")
       ↓
  Returns TRUE → grant access
```

## Troubleshooting

**"Database connection failed"**
- Ensure MySQL is running in XAMPP Control Panel.
- Check credentials in `config/db.php`.
- Confirm the database `hashguard_db` was imported successfully.

**"Page not found" (404)**
- Confirm the folder is named exactly `hashguard` inside `htdocs`.
- Visit `http://localhost/hashguard/` (with trailing slash).

**Login fails for admin account**
- The bcrypt hash in `database.sql` uses prefix `$2b$` (generated by Python bcrypt).
  PHP 7+ accepts this as valid. If you still have issues, register a new account,
  then run this SQL in phpMyAdmin to promote it to admin:
  ```sql
  UPDATE users SET role = 'admin' WHERE email = 'your@email.com';
  ```

**Styles not loading**
- Make sure you're accessing the site via `http://localhost/hashguard/` and NOT
  by opening the `.php` files directly in the browser.

## Resetting the Admin Password

In phpMyAdmin, run:

```sql
USE hashguard_db;

-- Replace the hash below with one generated by PHP:
-- <?php echo password_hash('YourNewPassword', PASSWORD_BCRYPT); ?>

UPDATE users
SET password = '$2y$12$REPLACE_WITH_YOUR_HASH'
WHERE email = 'admin@hashguard.com';
```

Or simply register a new account and promote it to admin as shown above.
